# 2026-06-03 开发记录

## 今日目标

- 为前端卡片式预览提供统一摘要字段。
- 扩充论文抓取数量和 AI 相关学科范围。
- 让 AI 前沿栏目同时包含科技资讯与论文。

## 1. 卡片摘要字段

问题：新闻接口只有 `description`，论文接口只有较长的 `abstract`，前端无法用统一字段渲染卡片摘要。

处理：

- 新增确定性的短摘要生成逻辑，清理 HTML、合并空白，并优先在句子边界截断。
- 新闻、论文和 AI 前沿接口条目统一返回 `summary`。
- 条目增加 `item_type`，值为 `news` 或 `paper`。
- Android 后端 DTO 优先读取 `summary`，并兼容原有描述和摘要字段。

结果：前端卡片可以直接使用 `summary`，无需自行判断新闻或论文的数据结构。

## 2. 论文内容补充

问题：论文抓取默认数量较少，`auto` 模式在 RSS 成功后不会再补充 API 的关键词论文。

处理：

- 论文每次抓取至少请求 100 条。
- arXiv 学科范围增加 `cs.IR`、`cs.NE` 和 `cs.RO`。
- `auto` 模式合并 RSS 与 API 结果，按论文 ID 去重并按发布时间排序。
- arXiv RSS 描述中的作者与摘要分离，卡片摘要不再包含 `Authors:` 前缀。

结果：论文池覆盖范围更广，关键词推荐和最新 AI 论文可以同时补充。

## 3. AI 前沿资讯补充

问题：AI 前沿接口只要存在论文，就不会返回科技资讯，栏目内容结构单一。

处理：

- 新增 AI 科技资讯筛选，覆盖人工智能、大模型、智能体、多模态、机器人及主流 AI 产品关键词。
- AI 前沿接口交错返回科技资讯与论文，避免其中一种内容完全遮住另一种。
- AI 前沿论文不受学术推荐自选关键词限制，保持栏目定位独立。
- 新闻来源增加量子位和 TechCrunch AI 官方 RSS。
- App 刷新 AI 前沿时同时触发新闻与论文抓取。

结果：AI 前沿栏目能够持续展示行业资讯和学术论文两类内容。

## 4. arXiv 论文主图

问题：arXiv RSS 和 API 只提供标题、作者、摘要等元数据，不提供论文中的主要图像，论文卡片和预览缺少直观信息。

处理：

- 新增 arXiv 官方 HTML 论文页面解析器，读取正文 `figure` 中的图片与图注。
- 从正文前几张候选图中选择尺寸较大的主要图像，过滤 logo、icon 等装饰图片。
- SQLite `papers` 表自动增加 `image_url`、`image_caption` 和 `figure_checked_at` 字段。
- 新增独立主图富化命令和接口，避免每次论文抓取都逐篇访问 HTML 页面。
- Android 论文卡片复用现有图片显示能力，论文正文预览中附带图注。

使用命令：

```powershell
python -m backend.topnews_backend.cli paper-figures-ingest --limit 10 --delay-seconds 3
```

结果：成功富化后的论文会在推荐接口中返回主要图片和图注；没有 HTML 版本或可用图像的论文仍可正常展示。

## 5. 论文图片 SQLite 缓存

问题：真机所在网络可能无法直接访问 arXiv 图片地址，即使论文接口已经返回 `image_url`，卡片仍会显示灰色占位。

处理：

- `papers` 表自动增加图片 BLOB、MIME 类型和缓存时间字段。
- 主图富化任务在提取图片地址后，将图片二进制下载并保存到 `topnews.db`。
- 已经保存远程图片 URL、但还没有图片 BLOB 的旧论文会自动重新进入待补缓存队列。
- 新增后端图片接口 `GET /v1/papers/{external_id}/image`，从 SQLite 输出图片内容。
- 论文推荐接口的 `image_url` 改为后端图片接口地址，并通过 `image_source_url` 保留原始 arXiv 地址。
- Android 客户端会把相对图片地址解析为当前 TopNews 后端地址。

结果：论文图片展示只依赖真机已经能够访问的 TopNews 后端，不再依赖真机直接连接 arXiv 图片域名。

## 6. 预览卡片图片与交互修复

问题：

- 主页缩略图可以显示，但打开半屏预览后图片区域保持灰色。
- 半屏预览打开时，用户仍然可以点击或滑动后方的栏目栏。

处理：

- 移除预览打开时没有尺寸限制的图片预加载，避免论文原图按完整尺寸解码失败。
- 预览内容使用稳定的全尺寸布局，避免入场动画开始时图片处于零尺寸而不触发加载。
- 预览图片只由受卡片布局尺寸约束的 `AsyncImage` 加载。
- 半透明遮罩层消费卡片外的完整触摸手势，阻止点击和横向滑动继续传递到主页。
- 主页栏目点击、横向分页和刷新入口在预览打开时同步禁用，作为交互层的额外保护。

结果：预览卡片能够复用主页已加载的图片，预览打开时后方主页不再可操作。

## 7. 论文图表完整显示

问题：论文主图在 arXiv HTML 中内容完整，但 App 预览卡片会缺少图表标题、横轴标签或坐标轴标识。

原因：预览与详情中的图片使用 `ContentScale.Crop` 填满固定区域，论文图表上下边缘会被裁剪。

处理：

- 半屏预览和全屏详情中的大图改为 `ContentScale.Fit`，完整显示图表标题、坐标轴和图例。
- 首页列表中的小缩略图继续保持裁剪显示，避免影响信息流布局。

结果：论文图表在阅读场景中不再被裁剪。

## 8. 图片宽高比自适应

问题：将论文图表改为完整显示后，固定高度的图片容器会在图表上下留下明显灰色空白。

处理：

- 图片加载成功后读取真实宽高比，并动态调整预览和详情中的图片容器比例。
- 移除半屏预览中的固定权重高度和详情页固定 `16:9` 比例。
- 对极端宽图和竖图设置合理比例边界，避免图片占满整个阅读区域。

结果：图片容器会匹配图表自身比例，完整显示内容的同时减少无意义留白。

## 9. 图片单击放大预览

处理：

- 半屏预览和全屏详情中的图片增加单击入口。
- 单击图片后以淡入和缩放动画打开全屏深色图片查看层。
- 放大预览保持图片完整比例，并显示文章标题作为上下文。
- 系统返回、右上角关闭或单击查看层可退出图片预览，文章预览状态保持不变。

结果：用户可以快速放大查看论文图表中的文字、坐标轴和细节。

## 10. 品牌配色统一

问题：应用主页仍使用默认 Material 紫色和新闻应用常见的红色交互色，与新版 Top News 图标的黄色、紫色、深蓝色视觉风格不一致。

处理：

- 新增品牌黄色、品牌紫色、品牌深蓝色以及背景、正文、次要文字、分割线等语义颜色令牌。
- 完善 Compose Material 亮色与暗色主题，并默认关闭 Android 动态取色，避免品牌色被设备壁纸覆盖。
- 主页头部改为品牌黄色，天气信息使用深蓝文字，AI 回答入口使用品牌紫色。
- 栏目选中状态、搜索入口、置顶标签、错误重试和视频播放标识统一使用品牌紫色。
- 新闻列表、预览卡片和详情页改用主题中的正文色、次要文字色、图片占位色与分割线色。
- Android 状态栏、导航栏和系统强调色同步使用品牌配色。

结果：应用视觉风格与新版图标保持一致，同时保留白色内容区和中性色层级，保证新闻阅读的清晰度。

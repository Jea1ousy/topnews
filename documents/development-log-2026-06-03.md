# 2026-06-03 开发记录

## 今日目标

- 为首页新闻列表增加点击交互。
- 点击新闻后先显示半屏内容预览。
- 用户继续向上拉时，将预览窗口切换为全屏新闻阅读页。
- 用户下拉关闭或点击返回后，回到原来的首页新闻列表。
- 统一半屏预览与全屏详情中的图片和正文顺序。

## 1. 新闻列表点击入口

问题：新闻列表项原本只负责展示标题、来源、时间和封面图，没有对外提供点击事件，因此首页无法进入预览或详情状态。

处理：

- `NewsListItem` 增加 `onClick` 回调，并将整个新闻列表项设置为可点击区域。
- `NewsList` 增加 `onArticleClick` 参数，将点击的 `NewsArticle` 传递给首页。
- `HomeScreen` 保存当前选中的新闻，并以此控制新闻预览窗口的显示。

结果：用户可以点击任意新闻列表项，进入该新闻的内容预览。

## 2. 半屏新闻预览

处理：在 `HomeScreen` 中引入 Compose Material 3 `ModalBottomSheet`，并启用部分展开状态。点击新闻后，底部窗口以半屏形式显示以下内容：

- 新闻来源和发布时间。
- 新闻标题。
- 新闻封面图或视频封面。
- 新闻正文或摘要的前几行。

半屏状态保留圆角和拖拽把手，让用户能够感知该窗口可以继续向上拉或向下关闭。

结果：用户不离开首页即可快速判断新闻内容是否值得继续阅读。

## 3. 上拉进入全屏阅读

问题：仅将底部窗口高度增加到接近全屏，视觉上仍然像一个抽屉，不符合完整新闻阅读页的体验。

处理：

- 监听 `ModalBottomSheet` 的 `Expanded` 状态。
- 窗口完全展开时占满可用屏幕，并移除顶部圆角和拖拽把手。
- 全屏状态切换为独立的新闻详情排版，包括顶部返回、搜索、听和更多操作区域，来源信息、标题、正文阅读区，以及底部分享、评论、点赞和收藏操作栏。
- 顶部状态栏和底部导航栏分别使用 `statusBarsPadding` 与 `navigationBarsPadding` 适配全面屏设备。

结果：用户将半屏预览继续上拉到顶后，会获得接近常见新闻客户端的全屏阅读体验。

## 4. 返回首页列表

处理：

- 用户向下拖动并关闭底部窗口时，清除当前选中的新闻。
- 全屏详情页左上角返回入口会主动隐藏窗口，并回到原来的首页列表。
- 首页列表和栏目状态不会因为打开详情而重新创建。

结果：用户可以在预览、详情和首页列表之间连续切换。

## 5. 预览与详情格式统一

问题：初版全屏详情将封面图放在正文末尾，而半屏预览将图片放在正文前，导致同一篇新闻在两个状态下的阅读顺序不一致。

处理：半屏预览和全屏详情统一采用“来源信息、标题、封面图、正文”的内容顺序。

结果：从预览上拉进入详情时，文章结构保持一致，视觉切换更自然。

## 6. 模拟器后端连接说明

Android Studio 模拟器中的 `127.0.0.1` 指向模拟器自身，无法直接访问宿主机启动的 TopNews 后端。模拟器开发时，App 的后端地址应配置为：

```properties
TOPNEWS_BACKEND_BASE_URL=http://10.0.2.2:8080/
```

后端仍可在宿主机使用以下命令启动：

```powershell
python -m backend.topnews_backend.cli serve --host 127.0.0.1 --port 8080
```

修改 `local.properties` 后需要重新构建并安装 App，因为该地址会编译进 `BuildConfig`。`local.properties` 属于本机配置，不提交到仓库。

## 7. 当前实现边界

- 全屏阅读页目前由同一个 `ModalBottomSheet` 的完全展开状态承载，还没有引入独立导航路由。
- 分享、评论、点赞、收藏、关注、搜索和听新闻入口目前只完成视觉布局，尚未接入真实业务逻辑。
- 正文优先使用 `NewsArticle.content`，为空时回退到 `description`，两者都为空时显示暂无正文提示。
